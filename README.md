# Система автоматической оценки ликвидности земельных участков ИЖС

## Общее описание

Система предназначена для автоматической оценки ликвидности и рыночной стоимости земельных участков категории ИЖС на основе открытых данных и машинного обучения.

### Целевая аудитория
- Физические и юридические лица
- Инвесторы и застройщики
- Банки и оценочные компании

### Основные функции
- Поиск участка по кадастровому номеру, координатам или адресу
- Сбор и агрегация данных из различных источников
- Вычисление признаков на основе геоданных и инфраструктуры
- Расчет индекса ликвидности и прогноз рыночной стоимости
- Генерация отчета с объяснением факторов (Explainable AI)
- Визуализация на карте
- Сравнение с аналогичными участками

### Архитектурные решения

**Микросервисная архитектура** - выбрана для:
- Независимого масштабирования компонентов
- Раздельной разработки команд
- Гибкого выбора технологий для каждого сервиса
- Повышенной отказоустойчивости

**Облачная инфраструктура** - рекомендуется:
- AWS/GCP/Azure для гибкости и масштабируемости
- Kubernetes для оркестрации контейнеров
- S3/Google Cloud Storage для хранения сырых данных

## Технологический стек

### Backend
- **Python 3.10+** - основной язык программирования
- **FastAPI** - для создания REST API
- **SQLAlchemy + Alembic** - ORM и миграции
- **Celery + Redis** - асинхронные задачи
- **Apache Airflow** - оркестрация ETL процессов

### Базы данных
- **PostgreSQL + PostGIS** - основное хранилище структурированных данных
- **MongoDB** - хранение сырых данных и документов
- **Redis** - кэширование и брокер сообщений

### Машинное обучение
- **CatBoost/LightGBM** - градиентный бустинг
- **scikit-learn** - классические ML алгоритмы
- **SHAP** - объяснение моделей
- **MLflow** - управление ML экспериментами

### Геоданные
- **GeoPandas** - работа с векторными геоданными
- **Shapely** - геометрические операции
- **rasterio** - работа с растровыми данными
- **OSRM** - маршрутизация

### Frontend
- **React + TypeScript** - SPA фронтенд
- **Leaflet/OpenLayers** - картография
- **Chart.js/Plotly** - визуализация данных

### Инфраструктура
- **Docker + Kubernetes** - контейнеризация и оркестрация
- **Prometheus + Grafana** - мониторинг
- **Nginx** - балансировка нагрузки

## План разработки

1. **Этап 1: Подготовка (2 недели)**
   - Настройка CI/CD
   - Создание базовой инфраструктуры
   - Разработка схемы данных

2. **Этап 2: Core Services (4 недели)**
   - Реализация API Gateway
   - Разработка Data Service
   - Создание базы данных

3. **Этап 3: ML Pipeline (3 недели)**
   - Сбор и подготовка данных
   - Разработка ML модели
   - Интеграция SHAP

4. **Этап 4: Frontend (3 недели)**
   - Разработка веб-интерфейса
   - Интеграция с API
   - Визуализация данных

5. **Этап 5: Деплой и тестирование (2 недели)**
   - Настройка Kubernetes
   - Нагрузочное тестирование
   - Мониторинг и логирование

## Безопасность

- Валидация всех входных данных
- Ограничение частоты запросов (rate limiting)
- Шифрование чувствительных данных
- Аутентификация через API-ключи
- HTTPS для всех соединений

## Стоимость и производительность

**Оценочные затраты:**
- Разработка: 4-6 месяцев командой из 4-5 человек
- Инфраструктура: $500-2000/месяц в зависимости от нагрузки
- Лицензии на внешние API: $100-500/месяц

**Производительность:**
- Время ответа API: < 2 секунды
- Пропускная способность: 1000+ запросов/сек
- Хранение данных: 100+ ГБ структурированных данных
